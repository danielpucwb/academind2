<name>Ingestion service â€” stream, hash, write, commit</name>

<files>app/services/ingestion_service.py</files>

<read_first>@app/core/paths.py, @app/services/hierarchy_service.py</read_first>

<action>

Implement `IngestionService` taking `Engine`/`Session` pattern consistent with HierarchyService injection. Inputs: authenticated-by-existence `(course_id/discipline_id or discipline UUID)`, `UploadFile`-like reader, claimed filename/content_type. Algorithm:  
1. Validate extension against `settings.allowed_extensions`.  
2. Resolve bucket via extension map (documents/videos/audios/web).  
3. Load discipline + course slug via HierarchyService/repo to compute target directory + `ensure_discipline_paths`.  
4. Stream chunks â†’ incremental SHA256; enforce `max_upload_bytes`.  
5. Choose sanitized unique on-disk basename (prevent collisions unrelated to hashes).  
6. Write file with exclusive create (`xb`) or temp rename atomicity.  
7. Insert catalogue row transactional; rollback FS if DB fails (delete written file).

Duplicate path: abort before overwrite; return typed `DuplicateArtefactError`.

Traversal: final resolved path MUST stay under disciplined root.

</action>
